# Smart Infra 完全ワークフロー

これは統合オーケストレーションスキルであり、その役割はシステム管理者およびプロセスのガイドとして機能することです。
トリガーされた場合、以下の完全なクラウドインフラ設計プロセスを調整し、手順に沿ってプロジェクトの完了を導く必要があります。

# プロセス

## Phase 1: 要件の明確化フェーズ
まず、ユーザーに尋ねて**プロジェクトの英語コード名 (Project Name)** を特定し、これを今回のタスクの環境分離識別子として使用します。
次に、クラウドアーキテクトの役割を演じ、`cloud-architect` スキルのガイドラインを使用してまたは参照して、十分で明確なアーキテクチャ要件が収集されるまで、ユーザーと簡潔なやり取りを行います。
最後に、決定された要件を要約ドキュメントとして作成し、まず `.tmp/requirements.md` に書き込み、その後 `mkdir -p output/[プロジェクトコード]/ && mv .tmp/requirements.md output/[プロジェクトコード]/requirements.md` に移動して保存します。

## Phase 2: コード生成フェーズ
要件が確定し保存されたら：
すぐに役割をインフラ自動化エンジニアに切り替え、前のフェーズで生成された `output/[プロジェクトコード]/requirements.md` を読み取り、内部の `iac-generate` スキルの生成ロジックに似た処理を実行します。
`output/[プロジェクトコード]/terraform/` ディレクトリに、仕様に従ってフォーマットされた一連のモジュール化された `.tf` ファイルを生成します。

## Phase 3: アーキテクチャの可視化フェーズ
Terraform ファイルの生成が完了したら：
そのアーキテクチャの Mermaid グラフと Python Diagrams グラフを同時に生成します。
- Mermaid は `output/[プロジェクトコード]/diagrams/architecture.md` に書き込みます
- Python スクリプトは `output/[プロジェクトコード]/diagrams/generate.py` に書き込み、即座に実行して `architecture.png` を生成します。

出力が完了したら、**チャットボックスではグラフがレンダリングされない可能性があるため、ユーザーに必ず `architecture.md` ファイルをクリックして開いて可視化プレビューを行うようリマインドしてください。**

## Phase 4: ユーザー確認とイテレーション
提示の段階で一旦停止し、必ずユーザーに以下の点を確認してください：
1. 「生成されたアーキテクチャ図を確認してください。追加の要望や変更したい点はありますか？（例：コンポーネントの追加、スペックの変更など）自然言語で教えていただければ、こちらでコードと図を修正します。」
2. ユーザーからフィードバックがあった場合は、内部でTerraformコードを修正し、再度Phase 3（可視化）を実行して更新版の図を提示します。Terraformの知識がなくても設計が進められるよう配慮してください。

## Phase 5: 最終納品と操作案内
ユーザーが構成に満足し、追加の変更がないことを確認できたら、以下の内容を伝えて締めくくります：

- **成果物の場所**:
  - ✅ 要件定義書: `output/[プロジェクトコード]/requirements.md`
  - ✅ アーキテクチャ図: `output/[プロジェクトコード]/diagrams/`
- **デプロイの準備と実行**:
  「インフラの設計が完了しました。AWS上に実際にリソースを作成するには、以下の**事前準備**を完了させる必要があります：」

  1. **AWS CLI のセットアップ**: `aws configure` を実行して、認証キーを設定してください。
  2. **権限の確認**: 使用する AWS ユーザーにリソース作成権限があることを確認してください。

  「準備が整いましたら、以下のコマンドを順番に実行してください：」
  ```bash
  cd output/[プロジェクトコード]/terraform
  terraform init    # 構成の初期化
  terraform plan    # 作成内容のシミュレーションと確認
  terraform apply   # クラウドへの実際の反映（途中で 'yes' を入力）
  ```
  「認証の設定方法や各コマンドの詳細について不安がある場合は、いつでも私にお尋ねください。」
