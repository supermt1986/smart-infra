# Smart Cloud Infra (Agent Skills Workflow)

このリポジトリは、自動化されたクラウドインフラストラクチャ設計と Infrastructure as Code (IaC) 生成のための **AI エージェントスキル (Agent Skills) ワークフロー** を提供します。

従来のローコード (Low-Code) プラットフォームをフルスクラッチで開発する代わりに、このプロジェクトは Claude Code / OpenCode のような AI コーディングエージェントに直接スキルを付与し、自然言語によるインタラクティブなクラウドアーキテクチャ設計を可能にします。

## 🌟 主な機能

- **クラウドアーキテクトによる要件明確化**: AI がシニアクラウドアーキテクトとして振る舞い、対話を通じてユーザーの要件（高可用性、トラフィック規模、予算、セキュリティ要件など）を引き出します。
- **自動 IaC 生成**: 明確化された要件に基づき、AWS / GCP / Azure 向けの本番レベルの Terraform コードを自動生成します。
- **アーキテクチャの可視化**: 生成された Terraform 構成を、即座に Mermaid クイックプレビューおよび Python `diagrams` ライブラリを用いたプロフェッショナルな画像 (PNG) にレンダリングします。

## 📁 提供される Skills

すべてのスキルは `.agents/skills` ディレクトリに標準フォーマットで配置されています：

1. `cloud-architect`: プロジェクト要件の段階的な明確化を支援し、`requirements.md` を出力します。
2. `iac-generate`: 要件定義書に基づき、マルチクラウドで動作する Terraform `.tf` 構成をモジュール別に生成します。
3. `arch-diagram`: IaC コードを解析し、アーキテクチャの視覚的ドキュメント (`architecture.md` と `architecture.png`) を生成します。
4. `infra-flow`: 上記 1 〜 3 の統合オーケストレーションワークフロー。対話から図の確認・コード出力までワンストップで実行します。

## 🚀 クイックスタート

### 前提条件
手動で依存パッケージをインストールする必要はありません！
システムに **Python 3.x** と **Terraform CLI** 実行環境があれば、画像を描画するためのシステム依存（`pip install diagrams` や `graphviz` など）は、各 Agent がタスク実行時に自動的にチェックしてインストールします。

### アプリケーションの実行方法

ご使用の AI コーディング エージェント（Claude Code または OpenCode）を使用して、このリポジトリを作業ディレクトリとして開き、以下のようにコマンドまたは自然言語で指示を出してください：

**ワンストップ (推奨):**
> "infra-flow スキルを実行して、社内向けの高可用性従業員管理システムのクラウドインフラを設計して。"

**ステップバイステップ:**
> 1. "cloud-architect スキルを使用して、新しいEコマースサイトのインフラ要件を明確にして。"
> 2. "iac-generate を実行して、先ほど作成した要件から Terraform スクリプトを生成して。"
> 3. "arch-diagram を実行して、そのスクリプトのアーキテクチャ図を出力して。"

すべての実行成果物（要件定義書、Terraform 構成ファイル、および図）は、プロジェクト名単位で分離され、`output/[プロジェクトコード]/` フォルダ下に自動的に生成されます（※ `output/` 以下の実データは `.gitignore` によりバージョン管理から除外されます）。

## 📝 テンプレートと拡張

一般的なクラウド設計ベストプラクティスを AI に学習または参照させたい場合は、参考となる Terraform ファイルを `templates/terraform/common-patterns/` フォルダに追加してください。`iac-generate` スキルが自動的に参照し、品質を向上させます。
